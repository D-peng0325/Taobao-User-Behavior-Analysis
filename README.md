基于百万级电商行为的数据仓库重构与全链路增长增长看板

项目概述：
本项目针对百万级（100万+行）淘宝用户全链路行为数据集（PV、Fav、Cart、Buy），模拟企业级生产环境进行数据清洗、数仓分层建模、指标工程建设，并最终通过 Tableau 实现輕量化前端交付。旨在解决大数据量下商业指标查询缓慢的痛点，并为业务方提供高交互性的精细化运营决策方案。

核心代码目录
data_cleaning.py: 基于 Python (Pandas) 的原始数据清洗、异常值剔除与时间戳重构脚本。
taobao_analysis.sql: 基于 MySQL 的数仓 DWD/ADS 层视图封装、复合索引调优及核心业务指标高阶查询。

技术栈与工程亮点
数仓性能优化: 通过对 User_id, Date, Behavior_type 建立复合索引，遵循WHERE先过滤再GROUP BY原则，使百万级数据跨表聚合性能提升 80% 以上。
逻辑后置解耦: 运用 CTE 嵌套窗口函数动态计算全局均值，将 RFM 用户价值分层分支逻辑下沉并封装为数据库视图 (VIEW)，实现前端 BI 调用的“单一事实来源”。
商业全链路闭环: 独立多源驱动交付 大盘转化漏斗 与 同期群留存热力矩阵，成功诊断出全站进店购买转化率仅为 2.6% 且加购后流失严重的业务瓶颈。

核心 SQL 场景落地
本仓库的 .sql文件中包含以下大厂高频业务场景的硬核 SQL 解题方案：
1.  全站基础活跃与人均购买指标统计
2.  用户分时行为洞察（24小时加购/购买趋势下钻）
3.  全站爆款商品 Top 10 榜单
4.  忠诚用户挖掘（利用 Self-Join 自连接与 HAVING 状态计数实现）
